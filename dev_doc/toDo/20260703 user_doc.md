创建一个user_doc文件夹，在根目录下。

它的内容是用户使用时需要了解的信息（和开发者需要了解的信息不同）

具体有：
1. 各个 xxx_com.py文件在workflow.py里的详细用法。
2. calc_cost.py的写法
3. workflow的写法
4. 用户的工作流程：
   .1 准备好仿真文件，比如xxx.aedt。放到job_template里
   .2 准备好parameters_constraints.py。对于hfss，parameters可以用project/tools/hfss_get_para_and_range_direct.py自动化获取，constraints仍需要用户自己写.
   .3 从com_lib选出需要的 _com.py，复制到 job_template.py 文件夹里。
   .4 编写calc_cost.py和workflow.py。
   .5 修改project/config.py。
   .6 运行优化，使用start_optimization_aedtopt.cmd。正式运行优化之前最好运行一次smoke test.
