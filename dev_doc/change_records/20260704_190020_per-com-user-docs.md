# 2026-07-04 19:00 - Per-Com User Documentation

## Context

- The initial `user_doc` structure used one shared adapter document for all
  `_com.py` files.
- That shape would not scale well as more simulator adapters are added for
  different external software.

## Change

- Added `user_doc/com_lib/README.md` as the adapter documentation index and shared
  activation-rule guide.
- Added one document per current adapter:
  - `user_doc/com_lib/hfss_com.md`
  - `user_doc/com_lib/test_com.md`
- Updated `user_doc/README.md`, `optimization_workflow.md`, and
  `workflow_typical_patterns.md` to point to the per-adapter docs.
- Updated the `dev_doc` architecture file lists and previous user-doc change record
  to reference the new `user_doc/com_lib/` structure.
- Removed the old single adapter document `user_doc/com_files_in_workflow.md`.

## Rationale

- Future adapters such as Maxwell, TwinBuilder, or custom external tools can now add
  one focused `user_doc/com_lib/<adapter>.md` without making a single large document
  harder to navigate.

## Impact

- Users and AI assistants now read `user_doc/com_lib/README.md` first, then the
  specific adapter document needed by the task.
- New `_com.py` files should be documented by adding a matching Markdown file under
  `user_doc/com_lib/`.
