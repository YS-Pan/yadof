# 2026-07-18 21:00 - Package Run CLI

## Context

- Starting or resuming optimization still required root launchers even though the
  underlying optimizer and evaluators had moved into the package.

## Change

- Copied the launcher behavior into `yadof.run_command` and exposed `yadof run`
  with explicit workspace, generation/start index, mode, population, seed, smoke,
  progress, and all-infinite controls.
- The command loads workspace configuration once per generation, performs the
  optional exactly-one-individual smoke before generation zero, records progress,
  supports direct starts and history-backed resumes, and reports recent job
  diagnostics on failure.
- Removed both root launcher files and added direct/resume, smoke gate, override,
  progress, and diagnostic CLI tests.

## Rationale

- A package console command works from any directory and preserves one authoritative
  implementation of launch policy.

## Impact

- Users start and resume generic optimization with `yadof run --workspace PATH`.
- No repository root or source launcher is part of the runtime contract.
