# 2026-07-13 16:10 - Retire one-off tools diagnostics

## Context

`project/tools/` retained several diagnostics written for individual Windows
workstations and short-lived HTCondor investigations.  They obscured the reusable
data viewers, parameter generators, multicore diagnostic, and pool setup tools.

## Change

Moved these obsolete files to the Windows Recycle Bin:

- `collect_desktop2094_diagnostics.ps1`
- `fix_desktop2094_ansys_settings.ps1`
- `submit_desktop2094_ansys_probe.py`
- `submit_condor_ansys_hash_probe.py`
- `submit_condor_event_probe.py`
- `run_hfss_get_para_and_range_direct.bat`
- `htcondor_pool/setup_machine_worker.cmd`

The hash probe referenced a collector script that was no longer in the repository.
The event probe's diagnostic capability is now embedded in
`hfss_condor_multicore_diagnose.py`.  The Desktop-2094 scripts targeted one named
workstation and included a one-time registry repair.  The parameter-extraction
batch wrapper hard-coded the retired `AedtOPT` Conda environment, and the generic
worker wrapper was byte-for-byte identical to the machine-3 wrapper.

## Rationale

Keep reusable tools that operate on current recorded data, generate task parameter
definitions, run controlled multicore diagnostics, or configure/inspect the
documented HTCondor pool.  Retiring narrow probes avoids suggesting that a normal
deployment requires specific historic workstations or ad-hoc registry changes.

## Impact

Core modules and tests remain independent of tools.  The remaining parameter
generator is invoked directly as documented in `user_doc/optimization_workflow.md`.
The pool's supported manager/worker setup wrappers and generic configuration
scripts remain available.

## Follow-Up

None.
