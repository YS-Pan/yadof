创建一个user_doc文件夹，在根目录下。

它的内容是用户使用时需要了解的信息（和开发者需要了解的信息不同）。

具体有：
1. 各个 xxx_com.py文件在workflow.py里的详细用法。
2. calc_cost.py的典型写法
3. workflow的典型写法
4. 用户的工作流程：
   .1 准备好仿真文件，比如xxx.aedt。放到job_template里
   .2 准备好parameters_constraints.py。对于hfss，parameters可以用project/tools/hfss_get_para_and_range_direct.py自动化获取，constraints仍需要用户自己写.
   .3 从com_lib选出需要的 _com.py，复制到 job_template.py 文件夹里。
   .4 编写calc_cost.py和workflow.py。
   .5 修改project/config.py。
   .6 运行优化，使用start_optimization_aedtopt.cmd。正式运行优化之前最好运行一次smoke test.


user_doc需要有一个README.md，这个文档主要是给ai看的，因为很多工作是由用户指示ai执行的。它的风格类似于dev_doc/README.md。它的内容只包括用户需要知道的信息，所以内容上和dev_doc/README.md不同。
user_doc里应该有很多个文档，由它的readme来指示需要读取哪些文档。

编辑dev_doc/README.md，在其中添加 “读取user_doc/README.md，并按照它的指示读取文件”的命令。读取user_doc时不应该读取dev_doc，但读取dev_doc时必须读取user_doc。
user_doc 中已经出现了的内容，dev_doc 中不应该有重复。对于同一个文件或者模块，如果两边有不同的视角，可以各自写一个文档。但如果内容几乎一样，则应当只保留user_doc中的那一份。
