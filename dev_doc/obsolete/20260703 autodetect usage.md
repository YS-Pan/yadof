自动化的配置job需要的资源数量。

当前的distributed job需要的资源是由用户手动填写的。
在config.py里有：
```
HTCONDOR_REQUEST_CPUS = 4
HTCONDOR_REQUEST_MEMORY = "8GB"
HTCONDOR_REQUEST_DISK = "5GB"
```
我希望将它改成自动化的。

具体有：
1. 首先运行smoke test，此时使用某个默认的资源需求量。
2. 运行时记录实际使用的资源量，放到metadata里，回传回来。
3. 根据回传的实际使用量，修改config.py里的需求量。对于memory和disk，应该乘一个系数，比如2（只有smoke test到第一代的时候用这个系数，代与代之间的规则参考后面的描述）。
4. 开始第一代仿真，用上述的实际使用量（乘系数）作为htcondor里申请的资源量。每个仿真也记录实际使用量。
5. 下一代的内存和硬盘申请量（cpu申请量不用这个规则），基于上一代的实际使用量。用去掉使用量最多的5%（设为一个参数）的结果后的最大值（第95%大的实际使用量）。这会导致5%的任务因为使用量超标而报错，为了避免完全的报错，condor job的内存和硬盘申请量的写法，应该参考“temp\condor 资源.md”里的内容，每次超额报错后翻倍。
6. 之后的每代都如此往复。

这里提到的倍率之类的参数放到 config all里。
