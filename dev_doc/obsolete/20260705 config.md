# 修改config.py.

现在的 project\config.py 过于杂乱，里面既有重要的东西，又有很细节的几乎永远不会修改的东西。甚至还有一段函数。
我希望修改并拆分 config.py。
具体为：
1. workflow.py里的这些参数要改为在config.py里进行设置，在workflow里面引用。
```
DEFAULT_JOB_CPUCORE = 6
DEFAULT_PARALLEL_TASKS = 1
DEFAULT_NON_GRAPHICAL = True
```

2. evaluate manager 里的 config.py 里面，所有有可能修改的变量，都应该引用自全局的config.py。类似PROJECT_DIR这种不需要修改的变量不要引用。

3. 如果现在的project\config.py里有2里提到的不需要修改的变量，请把它们迁出到对应的模块里。

4. 去掉现有的config.py中的函数，它应该只包含设置。dev_doc\blueprints\10_modules\config.md 应该写明不许有函数

5. 将config.py 拆分成两个。一个是包含全部config的文件，里面对于不同类型的配置应该有分隔的符号。另一个是关键配置，只包含少数几个（比如10个以内）的，对于每次优化都很可能需要修改的配置。
